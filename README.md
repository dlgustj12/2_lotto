1. 구현 목표 : 

예외 처리: IllegalArgumentException 대신 의미 있는 커스텀 예외 사용

2. 기능 요구 사항
로또 게임 규칙
로또 번호의 숫자 범위는 1~45까지이다.

로또 1장의 가격은 1,000원이다.

로또 구입 금액을 입력하면 해당 금액만큼 로또를 자동 발행한다.

1개의 로또는 중복되지 않는 6개의 숫자를 뽑는다.

자동 생성된 당첨 번호 6개와 보너스 번호 1개는 서로 중복되지 않아야 한다.

당첨 기준 및 결과
등수	일치 기준	상금
1등	6개 번호 일치	
2,000,000,000원 

2등	5개 번호 + 보너스 번호 일치	
30,000,000원 

3등	5개 번호 일치	
1,500,000원 

4등	4개 번호 일치	
50,000원 

5등	3개 번호 일치	
5,000원 


사용자가 구매한 로또 번호와 당첨 번호를 비교하여 당첨 내역 및 수익률을 출력하고 게임을 종료한다.

수익률은 소수점 둘째 자리에서 반올림하여 출력한다.

3. 프로그래밍 요구 사항 및 구현 전략
입력 및 예외 처리
로또 구입 금액은 1,000원 단위로 입력받으며, 나누어 떨어지지 않는 경우 예외 처리한다.

사용자가 잘못된 값을 입력할 경우 커스텀 에러를 발생시키고, [ERROR]로 시작하는 에러 메시지를 출력 후 종료한다.

예외 상황 시 IllegalArgurnentException를 직접 사용하지 않고, 의미에 맞는 커스텀 예외를 사용한다.


1. 1~45 범위를 벗어난 경우.

2. 1,000원 단위가 아닌 경우.

3. 숫자가 중복된 경우

예외 메시지는 [ERROR]로 시작해야 하며, 이 요구사항은 커스텀 예외의 생성자에서 처리한다.

출력
발행한 로또 수량과 번호를 출력하며, 로또 번호는 오름차순으로 정렬하여 보여준다.

당첨 내역을 출력한다.

4. 패키지 및 클래스(책임)
application--
Application	프로그램의 진입점 및 최종 예외 처리 담당.
view--
InputView	사용자 입력(구입 금액, 당첨 번호)을 받아 전달하는 책임.
OutputView	발행 정보, 당첨 통계, 수익률 등을 출력하는 책임.
domain--
Lotto	구매한 로또 1장 (6개 번호) 관리 및 번호 유효성 검증.
Winning	당첨 번호 6개와 보너스 번호 1개 관리 및 등수 판별 로직 구현.
Rank (Enum)	1~5등의 당첨 기준(일치 개수, 보너스 일치)과 상금 정의 및 주어진 조건에 맞는 등급 반환.
PurchaseAmount	구입 금액 관리, 1,000원 단위 검증, 로또 수량 계산.
exception--
ErrorCode (Enum)	순수 오류 메시지 정의 및 관리.
Exception	모든 커스텀 예외의 부모 클래스. 생성자에서 [ERROR] 접두사를 추가하는 로직 구현.
NumberException
PurchaseException
